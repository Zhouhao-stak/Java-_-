## 3. 需求分析

### 3.1 用户模块
| 功能点 | 需求描述 |
| --- | --- |
| **用户登录** | 输入用户名与密码，验证后进入系统界面；验证失败则显示错误 |

| 功能点 | 需求描述 |

## 4. 概要设计

系统采用**分层设计**，将系统分为界面层(UI)、业务逻辑层(Service)、数据访问层(DAO)、常量配置层(JDBCTools)。各层职责单一、低耦合，便于维护和扩展。

| 层级 | 类名 | 核心职责 |
| --- | --- | --- |
| **常量配置层** | `JDBCTools` | 定义数据库连接参数、界面样式常量（颜色、字体） |
| **界面层** | `UI` | 负责登录/主界面的绘制、组件事件绑定、用户输入获取、消息提示 |
| **数据访问层** | `BookSystemDAO` | 封装数据库操作（CRUD），提供与数据库的交互接口 |
| **业务逻辑层** | `BookSystemService` | 协调 UI 层和 DAO 层，处理业务规则（如参数校验、流程控制） |

---

## 5. 核心类设计说明

### 5.1 `JDBCTools` (常量类)
- **核心**：统一管理数据库连接参数和界面样式常量，采用静态常量 + 私有构造方法，便于全局调用和配置修改。

### 5.2 `UI` (界面层)
- **核心**：依赖注入 Service 层对象，仅负责界面渲染和用户输入传递，不处理业务逻辑。
- 采用 Swing 布局管理器组合（`BorderLayout` + `GridBagLayout`），保证界面自适应。
- 封装通用组件（按钮、表格）样式和交互逻辑，提升代码复用性。

### 5.3 `BookSystemDAO` (数据访问层)
- **核心**：仅负责数据库 CRUD 操作，不处理业务逻辑和界面交互。
- 使用 `try-with-resources` 自动关闭数据库资源，避免内存泄露。
- 采用 `PreparedStatement` 防止 SQL 注入。

### 5.4 `BookSystemService` (业务逻辑层)
- **核心**：协调 UI 层和 DAO 层，处理业务规则（参数校验、流程控制）。
- 捕获 DAO 层异常，转换为用户友好提示。

## 6. 课程设计总结

本次图书管理系统以 Java Swing 为界面框架、MySQL 为数据库开发，采用分层架构设计思路，将系统拆分为常量配置、界面展示、数据访问和业务逻辑四层，各层各司其职，降低了代码耦合度。系统实现了用户登录注册、图书信息增删改查等核心功能，开发中运用 `PreparedStatement` 防止 SQL 注入，让操作更便捷。

不过系统仍有不足，比如用户密码未加密、无权限管理、数据查询未分页等。后续可引入数据库连接池提升性能，对密码加密增强安全性，补充权限控制等功能，让系统更贴合实际使用需求。
